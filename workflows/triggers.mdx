## HTTP / Webhooks

Pipedream generates a unique HTTPS endpoint for each HTTP trigger. Send any request to that URL — GET, POST, PUT, PATCH, DELETE — and the workflow runs immediately.

### Event shape

```javascript
steps.trigger.event = {
  method:  "POST",
  path:    "/",
  query:   { source: "stripe" },
  headers: {
    "content-type": "application/json",
    "x-stripe-signature": "t=1714000000,v1=...",
  },
  body: {
    type: "payment_intent.succeeded",
    data: {
      object: { id: "pi_3P...", amount: 4999, currency: "usd" },
    },
  },
};
```

### Validating webhook signatures

```javascript
import crypto from "crypto";

export default defineComponent({
  async run({ steps, $ }) {
    const sig     = steps.trigger.event.headers["x-stripe-signature"];
    const payload = steps.trigger.event.bodyRaw; // raw string body
    const secret  = process.env.STRIPE_WEBHOOK_SECRET;

    const expectedSig = crypto
      .createHmac("sha256", secret)
      .update(payload)
      .digest("hex");

    if (`sha256=${expectedSig}` !== sig) {
      throw new Error("Invalid webhook signature");
    }

    return steps.trigger.event.body;
  },
});
```

### Returning a custom HTTP response

By default, Pipedream responds immediately with `200 OK` and processes the workflow asynchronously. To return a custom response synchronously:

```javascript
export default defineComponent({
  async run({ steps, $ }) {
    const { challengeToken } = steps.trigger.event.body;

    // Respond to Slack's URL verification challenge
    await $.respond({
      status: 200,
      headers: { "content-type": "application/json" },
      body: { challenge: challengeToken },
    });
  },
});
```

<Note>
  Synchronous responses require enabling **Return a response from this workflow** in the HTTP trigger settings. The workflow must complete within 30 seconds.
</Note>

## Schedule

Run a workflow on a fixed interval or a cron expression. The trigger event contains the timestamp of the scheduled run.

### Intervals

Choose from preset intervals in the trigger settings:

- Every 15 minutes
- Every hour
- Every day
- Every week

### Cron expressions

Use a standard 5-field cron expression for fine-grained control:

| Expression | Meaning |
|---|---|
| `0 9 * * 1-5` | Weekdays at 9:00 AM UTC |
| `0 */4 * * *` | Every 4 hours |
| `30 6 1 * *` | 1st of each month at 6:30 AM UTC |
| `0 12 * * 1` | Every Monday at noon UTC |

### Using the scheduled timestamp

```javascript
export default defineComponent({
  async run({ steps, $ }) {
    // steps.trigger.event.timestamp — ISO 8601 string
    const runAt = new Date(steps.trigger.event.timestamp);
    const windowStart = new Date(runAt.getTime() - 60 * 60 * 1000); // 1 hour ago

    const response = await fetch(
      `https://api.example.com/events?since=${windowStart.toISOString()}&until=${runAt.toISOString()}`
    );
    return response.json();
  },
});
```

<Tip>
  All schedule times are in UTC. Convert to a local timezone in your step code using the `luxon` or `date-fns-tz` packages.
</Tip>

## App events

Pipedream maintains event sources for 2,800+ apps. An event source polls or subscribes to an external service and emits an event each time something new happens — a new GitHub issue, a new Airtable row, a new Stripe payment.

### How it works

1. Select an app trigger (e.g., **GitHub → New Issue**)
2. Connect your account and configure options (repo, labels, etc.)
3. Pipedream runs the source on your behalf and emits each new event as a discrete workflow execution

### Event shape example — GitHub new issue

```javascript
steps.trigger.event = {
  id: 2300000001,
  title: "Button click handler not firing in Safari",
  body: "Steps to reproduce...",
  html_url: "https://github.com/acme/frontend/issues/142",
  state: "open",
  labels: [{ name: "bug" }, { name: "browser-compat" }],
  user: { login: "mwilliams", avatar_url: "..." },
  created_at: "2026-03-15T14:22:00Z",
};
```

### Example: triage GitHub issues with AI

```javascript
import Anthropic from "@anthropic-ai/sdk";

export default defineComponent({
  props: {
    github: { type: "app", app: "github" },
  },
  async run({ steps, $ }) {
    const issue = steps.trigger.event;
    const client = new Anthropic();

    const message = await client.messages.create({
      model: "claude-opus-4-5",
      max_tokens: 256,
      messages: [{
        role: "user",
        content: `Classify this GitHub issue into one of: bug, feature-request, question, docs.\n\nTitle: ${issue.title}\nBody: ${issue.body}`,
      }],
    });

    return {
      issueNumber: issue.number,
      category: message.content[0].text.trim(),
    };
  },
});
```

## Email

Pipedream generates a unique inbound email address for your workflow. Emails sent to that address are parsed and delivered as structured events.

### Event shape

```javascript
steps.trigger.event = {
  from: [{ address: "sender@example.com", name: "Sender Name" }],
  to:   [{ address: "abc123@pipedream.net" }],
  subject: "Nightly export ready",
  text: "Your CSV export is attached.",
  html: "<p>Your CSV export is attached.</p>",
  attachments: [
    {
      filename: "export-2026-03-15.csv",
      contentType: "text/csv",
      size: 48392,
      content: "<base64-encoded content>",
    },
  ],
  headers: { "message-id": "<...@mail.example.com>" },
};
```

### Parsing an attachment

```javascript
import Papa from "papaparse";

export default defineComponent({
  async run({ steps, $ }) {
    const attachment = steps.trigger.event.attachments[0];

    if (!attachment || attachment.contentType !== "text/csv") {
      $.flow.exit("No CSV attachment found");
    }

    const csv = Buffer.from(attachment.content, "base64").toString("utf8");
    const { data } = Papa.parse(csv, { header: true, skipEmptyLines: true });

    return {
      filename: attachment.filename,
      rowCount: data.length,
      rows: data,
    };
  },
});
```

## Custom polling trigger

When no pre-built source exists for your use case, you can write a custom event source that polls an API on a schedule and emits only new items.

```javascript
import { axios } from "@pipedream/platform";

export default {
  name: "New Order from Custom API",
  version: "0.0.1",
  props: {
    http: "$.interface.http",
    db:   "$.service.db",
    timer: {
      type: "$.interface.timer",
      default: { intervalSeconds: 300 }, // every 5 minutes
    },
    apiKey: {
      type: "string",
      label: "API Key",
      secret: true,
    },
  },
  async run() {
    const lastId = await this.db.get("lastOrderId") || 0;

    const orders = await axios(this, {
      url: "https://api.example.com/orders",
      headers: { Authorization: `Bearer ${this.apiKey}` },
      params: { since_id: lastId, limit: 100 },
    });

    for (const order of orders) {
      this.$emit(order, {
        id:        order.id,
        summary:   `New order #${order.id} — $${order.total}`,
        ts:        Math.floor(new Date(order.created_at).getTime() / 1000),
      });
    }

    if (orders.length > 0) {
      await this.db.set("lastOrderId", orders[orders.length - 1].id);
    }
  },
};
```
