Pipedream is an integration platform for developers. Workflows are sequences of steps triggered by events — HTTP requests, schedules, or events from apps like GitHub, Slack, and Airtable. This guide walks you through building and deploying your first workflow.

<Note>
  Pipedream's free tier includes 10,000 events/month with no credit card required.
</Note>

## Prerequisites

- A Pipedream account — [sign up for free](https://pipedream.com/auth/signup)
- Basic familiarity with APIs or webhooks (helpful but not required)

## Build your first workflow

<Steps>
  <Step title="Sign up and open the workflow editor">
    Go to [pipedream.com](https://pipedream.com) and create a free account. Once logged in, click **New Workflow** in the top-right corner.

    You'll land in the workflow editor — a canvas where you add and configure steps. Every workflow starts with a trigger.
  </Step>

  <Step title="Add an HTTP trigger">
    Click **Add a trigger** and select **HTTP / Webhook**. Pipedream immediately provisions a unique endpoint URL for your workflow, such as:

    ```
    https://en1234abc.m.pipedream.net
    ```

    Any HTTP request sent to this URL will trigger your workflow. The trigger automatically parses the request method, headers, query parameters, and body, making them available to downstream steps as `steps.trigger.event`.

    <Tip>
      You can also trigger workflows on a schedule (cron), by email, or from app events like a new GitHub issue or Slack message.
    </Tip>
  </Step>

  <Step title="Add a code step">
    Click **+** below the trigger to add a new step, then select **Run custom code**. Choose your preferred language:

    <Tabs>
      <Tab title="Node.js">
        ```javascript
        export default defineComponent({
          async run({ steps, $ }) {
            const body = steps.trigger.event.body;
            console.log("Received payload:", body);
            return body;
          },
        });
        ```
      </Tab>
      <Tab title="Python">
        ```python
        def handler(pd: 'pipedream'):
            body = pd.steps["trigger"]["event"]["body"]
            print("Received payload:", body)
            return body
        ```
      </Tab>
      <Tab title="Go">
        ```go
        package main

        import (
            "fmt"
            pd "github.com/PipedreamHQ/pipedream-go"
        )

        func main(pdEvent pd.Event) interface{} {
            body := pdEvent.Steps["trigger"].(map[string]interface{})["event"]
            fmt.Println("Received payload:", body)
            return body
        }
        ```
      </Tab>
    </Tabs>

    Every step returns a value. Return values are exported as `steps.<step_name>.$return_value` and are accessible by all subsequent steps.
  </Step>

  <Step title="Connect a pre-built action">
    Click **+** to add another step, then search for an app — for example, **Slack**. Select the **Send a Message** action.

    Connect your Slack account when prompted. Pipedream stores your credentials securely and manages OAuth refresh automatically. Configure the action:

    - **Channel**: `#general`
    - **Message text**: You can reference previous step output directly: `{{steps.trigger.event.body.message}}`

    With 2,800+ integrated apps and 10,000+ pre-built components, you can connect almost any service without writing boilerplate auth or API code.
  </Step>

  <Step title="Deploy and send a test request">
    Click **Deploy** in the top-right corner. Your workflow is now live.

    Send a test request using `curl`:

    ```bash
    curl -X POST https://en1234abc.m.pipedream.net \
      -H "Content-Type: application/json" \
      -d '{"message": "Hello from curl"}'
    ```

    Replace the URL with your actual endpoint. You'll see the event appear in the **Inspector** panel on the right — click any step to inspect its input, output, and logs in real time.

    <Check>
      If your Slack action ran successfully, you'll see "Hello from curl" in your `#general` channel.
    </Check>
  </Step>
</Steps>
